# Agent Guidelines

- For medium-to-large tasks, include a todo list at the end that covers the remaining tasks not yet completed.
- If the current git branch is `master`, create a new working branch before making code changes unless the user explicitly asks to stay on `master`.
- Before making code changes, write or update a test first when practical, verify it fails for the expected reason, and then implement the code change.
- For bug fixes, write a test that reproduces the bug first, verify it fails for the right reason, and then implement the code fix.
